[SLIDE 3] 프로젝트 배경 — 왜 만들었나
────────────────────────────────────────────────────────────────────────────────

제목: 사이버 공격은 증가하는데, 탐지·분석 도구는 분산되어 있다

내용:
  ▸ 2024년 기준 사이버 공격 건수 전년 대비 38% 증가 (IBM X-Force)
  ▸ 허니팟은 공격자 전술·기술·절차(TTP) 수집에 효과적인 수단
  ▸ 그러나 기존 허니팟 도구들의 문제:
      - 단일 프로토콜만 커버 (SSH만, 웹만, ICS만...)
      - 로그 수집·분석이 수동 작업
      - 전문 지식 없이는 결과 해석 어려움
      - 공격 시뮬레이션과 탐지를 함께 할 수 없음

  → 다중 허니팟 + 자동 분석 + 시각화를 하나로 묶은 통합 플랫폼 필요

[SLIDE 4] 기존 솔루션 분석 — T-Pot
────────────────────────────────────────────────────────────────────────────────

제목: T-Pot: 강력하지만 진입장벽이 높다

내용:
  T-Pot이란?
    독일 Telekom Security가 개발한 오픈소스 허니팟 플랫폼
    30개 이상의 허니팟을 Docker로 한 번에 운영
    Elasticsearch + Kibana로 로그 시각화

  T-Pot의 강점:
    ✓ 허니팟 종류 다양 (30개+)
    ✓ 오픈소스, 활발한 커뮤니티
    ✓ Kibana 대시보드 제공

  T-Pot의 한계:
    ✗ 공인 IP가 있는 별도 서버 필요 (클라우드 비용 발생)
    ✗ 공격 시뮬레이션 기능 없음 (수동 대기)
    ✗ ML/AI 분석 미지원 — 로그 해석은 전문가 몫
    ✗ 학습/교육 목적으로 즉시 사용하기 어려움

  → 우리 플랫폼: T-Pot의 교육용·데모용 경량 버전으로 포지셔닝

[SLIDE 5] 우리의 접근 방식
────────────────────────────────────────────────────────────────────────────────

제목: T-Pot 구조를 참고해 로컬에서 즉시 실행 가능한 통합 플랫폼 구축

3단계 전략:

  [1단계: 로컬 통합 플랫폼] ← 현재 (1~8주차 완료)
      - T-Pot의 핵심 허니팟 7종을 Docker Compose로 로컬 구동
      - Kali 컨테이너로 공격 시나리오 자동화 (9가지)
      - ML + LLM으로 자동 분류 및 분석
      - 웹 대시보드로 비전문가도 사용 가능

  [2단계: AWS T-Pot 연동] ← 예정 (9주차~)
      - AWS EC2에 T-Pot HIVE 구축 (실제 인터넷 공격 수집)
      - 실데이터로 ML 모델 재학습 → 정확도 향상
      - 로컬 플랫폼 ↔ T-Pot 로그 파이프라인 연결

  [3단계: T-Pot 오픈소스 기여] ← 최종 목표
      - ML 분류 컨테이너 PR 기여
      - 한국어 Kibana 대시보드 기여

  공격 시나리오 9종:
    01 정상 트래픽        02 포트 스캔 (nmap)

    05 침투 후 명령 실행   06 역방향 셸
    07 악성코드 업로드     08 크리덴셜 스터핑
    09 ICS 공격 (Modbus)

  → 버튼 클릭 한 번으로 시나리오 실행 → 로그 자동 수집

────────────────────────────────────────────────────────────────────────────────

제목: RandomForest 기반 2단계 자동 분류

    dataset.csv    13,653행 / 29컬럼
    dataset_ml.csv 라벨 분포:
      Recon       5,402건 (39.6%)
      Etc         7,995건 (58.6%)
      Brute Force   220건 ( 1.6%)
      Intrusion      36건 ( 0.3%)

  분류 모델:
    1단계 (이진)   정상 vs 공격 판별
    2단계 (다중)   Recon / Brute Force / Intrusion / Malware / Etc

  MITRE ATT&CK 연계:
    각 공격에 T1046 / T1110 / T1190 등 전술 코드 자동 매핑
    점수 ≥ 70 → LLM 2차 분석 자동 트리거

[SLIDE 11] 핵심 기능 ③ — LLM 분석 & 프로파일링
────────────────────────────────────────────────────────────────────────────────

제목: 고위험 공격만 Ollama LLM이 자동 분석 → 공격자 프로파일 생성

  LLM 분석 흐름:
    공격 분류 결과 → MITRE 점수 계산
    점수 ≥ 70 → Ollama (gemma3) 자동 호출
    출력: 위험도 / 심각도 / 공격 요약 / 대응 방안

  Spring Boot 프로파일링:
    공격자 IP별 행동 이력 누적
    반복 공격 패턴 탐지
    PostgreSQL에 영속화

  대시보드 연동:
    D3.js 세계 지도 — 공격자 IP 실시간 시각화
    프로파일링 페이지 — 공격자별 상세 이력
